File Input/Output

  • 文件是数据存储的一种形式, 有两种 展现 形式: 文本文件和二进制文件.

  • 本质上, 所有文件都以二进制形式存储.

  • 形式上, 所有文件采用两种方式展示.

  • 两种方式可以用于所有数据, 但是有些文件天然就是文本文件, 有些文件更适合二进制形式.

  • 文本文件: 由 单一 特定编码组成的文件, 比如 utf-8 编码. 由于存在编码, 文本文件可以看成存储的长字符串.

  • 二进制文件: 直接由比特 0 和 1 组成, 没有统一 的字符编码, 根据预定义的 0-1 格式存储, 如 png, avi 等.

  • 所有文件都可以用二进制形式打开/读取.

  • 使用文本文件时, 如果需要理解其中的内容, 需要用文本格式打开. 如果只为了程序使用, 建议用二进制打开.

  • 文件的处理步骤: 打开 -> 操作 -> 关闭.

  • 文件本身是存储状态, 如果要进行操作, 需要打开文件使得文件处于 占用状态, 通过文件关闭可恢复 存储状态.

文件操作函数:

  • a.open(fname, mode) # 打开, 处于占用状态

  • a.read(size=-1) # size 为长度; 无参数则读入全部内容

  • a.readline(size=-1) # 读入一行的前 size 长度; 无参数则读入整行

  • a.readlines(hint=-1) # 读入前 hint 行; 无参数读入全部行; 按行返回 列表

  • a.write(s) # 写入字符串

  • a.writelines(lines) # lines 是元素为字符串的列表, 拼接 后无换行写入.

  • a.seek(offset) # 改变文件操作当前指针的位置. offset: 0-文件开头, 1-当前位置, 2-文件结尾

  • a.close()

7 种文件打开模式(可以进行不冲突组合):

  • r # 只读模式(默认). 文件不存在返回 FileNotFounderrorf

  • w # 覆盖写. 不存在则创建, 存在则 完全 覆盖

  • x # 创建写. 不存在则创建, 存在则返回 FileExistsError

  • a # 追加写. 不存在则创建, 存在则在文件后追加内容

  • b # 二进制文件模式

  • t # 文本文件模式(默认)

  • + # 可与 r,w,x,a 一同使用, 在原功能基础上增加同时读写功能

  • a.read() 一次读入, 统一处理(长字符串). 如果文件特别大, 这样处理不佳.

  • a.read(2) 可以每次处理小量的内容, 逐步处理. 适用于大文件.

  • 分行存储的文件采用逐行遍历的方法特别常见. readline() 适用于遍历读入处理, readlines() 适用于一次读入.

  • 文件操作中, 指针会随着一些操作不断流动, 需要 关注这一点.

  • 按行读入时, 每行最后存在 \n, 可以用 replace('\n', '') 将其消除.

例子:

fname = input("请输入要打开的文件名称: ")
tf = open(fname, 'rt')      # 'rt' 表示文本文件(text)
bf = open(fname, 'rb')      # 'rb' 表示二进制(binary)
-------------------
txt = tf.read()             # 一次读入, 统一处理(长字符串)
-------------------
txt = tf.read(2)
while txt != "":
    txt = tf.read(2)
-------------------
for line in tf:          ## 注意遍历集合使用文件打开句柄 tf
    print(line)
-------------------
tf.writelines(['中国', '法国', '美国'])
for line in tf:
    print(line)     # 此时不会打印任何信息
-------------------
tf.writelines(['中国', '法国', '美国'])
tf.seek(0)          # 指针指向文件开头, 便于打印输出
for line in tf:
    print(line)     
-------------------
tf.close()

自动化方式: 现实应用中将数据和功能分离, 通过文件操作方法建立两者的接口, 全程清晰明了.
